# 통신 오류 검출 방식 핵심 정리

## 1. 패리티(Parity)
- 데이터의 1 비트 개수를 짝수/홀수로 맞춰 오류 검출
- 1비트 오류만 검출 가능
- 구현 매우 단순, 오류 검출 능력 약함
- 현재는 교육·기초·특수 하드웨어 용도로만 사용

---

## 2. 블록합(Block Sum, 2D Parity)
- 행/열 단위로 패리티 비트를 추가하는 방식
- 패리티의 확장 형태
- 일부 오류 위치 추정 가능
- 실무보다는 개념 설명/교재용

---

## 3. 체크섬(Checksum)
- 데이터(보통 16비트 단위)를 **1의 보수 덧셈**으로 전부 더함
- 오버플로우 발생 시 **end-around carry**로 캐리를 다시 더함
- 최종 합의 **1의 보수**를 체크섬으로 전송
- 수신 측에서 데이터 + 체크섬을 다시 더해
    - 결과가 **0xFFFF**이면 정상
- 빠르지만 오류 검출 능력은 CRC보다 약함
- 사용처: **TCP, UDP, IP 헤더**

---

## 4. CRC (Cyclic Redundancy Check)
- 미리 약속된 **생성 다항식 G(x)** 사용
- 데이터에 대해 **mod-2 나눗셈(XOR)** 수행
- 나머지를 CRC 값으로 붙여 전송
- 수신 측에서 다시 나눠
    - 나머지 = 0 이면 정상
- 오류 검출 능력 매우 강함
- 사용처: **Ethernet, Wi-Fi, 디스크, 링크 계층**

---

## 핵심 비교 요약
- 패리티/블록합: 단순하지만 약함
- 체크섬: 빠른 소프트웨어 검증
- CRC: 강력한 하드웨어/링크 계층 검증
