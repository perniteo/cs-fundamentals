# OSI 7계층 정리

OSI(Open Systems Interconnection) 7계층은
네트워크 통신을 기능별로 나눈 **이론적 참조 모델**이다.

---

## 1️⃣ 물리 계층 (Physical Layer)

### 역할
- 비트(0,1)를 **전기/빛/전파 신호**로 변환
- 실제 물리적 전송 담당

### 특징
- 데이터 의미 ❌
- 주소 개념 ❌
- 오류 처리 ❌

### 데이터 단위
- Bit

---

## 2️⃣ 데이터링크 계층 (Data Link Layer)

### 역할
- **프레임 단위 전송**
- MAC 주소 기반 통신
- 오류 **검출** (수정 ❌)

### 주요 기능
- 프레임 생성 / 해제
- MAC 주소 처리
- CRC(FCS) 오류 검출

### 데이터 단위
- Frame

---

## 3️⃣ 네트워크 계층 (Network Layer)

### 역할
- **IP 주소 기반 경로 선택**
- 네트워크 간 패킷 전달

### 주요 기능
- 목적지 IP 확인
- 라우팅 (다음 홉 결정)
- TTL 감소
- (필요 시) IP 단편화

### 특징
- 신뢰성 ❌
- 순서 보장 ❌
- 상태 없음 (Connectionless)

### 데이터 단위
- Packet

---

## 4️⃣ 전송 계층 (Transport Layer)

### 역할
- **종단 간(end-to-end) 통신 보장**
- 데이터 흐름 관리

### 주요 기능 (TCP 기준)
- 세그먼트 분할
- 시퀀스 번호 부여
- 순서 보장
- 재전송 / 오류 복구
- 흐름 제어

### 프로토콜
- TCP (신뢰성)
- UDP (비신뢰성, 고속)

### 데이터 단위
- Segment

---

## 5️⃣ 세션 계층 (Session Layer)

### 역할
- **통신 세션 관리**

### 주요 기능
- 세션 설정
- 세션 유지
- 세션 종료
- 동기화 (체크포인트)

### 특징
- 데이터 전달 ❌
- 상태 관리 ⭕
- 현대에서는 응용 계층에 흡수됨

---

## 6️⃣ 표현 계층 (Presentation Layer)

### 역할
- **데이터 표현 방식 통일**

### 주요 기능
- 인코딩 변환
- 암호화 / 복호화
- 압축 / 해제

### 특징
- 데이터 의미 ❌
- 표현 방식만 변경
- 현대에서는 응용 계층에 흡수됨

---

## 7️⃣ 응용 계층 (Application Layer)

### 역할
- **응용 프로그램이 네트워크를 사용하도록 지원**
- 통신 규칙(프로토콜) 제공

### 주요 기능
- 요청/응답 처리
- 사용자 인터페이스 제공
- 응용 프로토콜 정의

### 대표 프로토콜
- HTTP
- FTP
- SMTP
- DNS

---

## 📌 계층별 데이터 단위 요약

| 계층 | 데이터 단위 |
|----|----|
| 응용 | Message |
| 표현 | Message |
| 세션 | Message |
| 전송 | Segment |
| 네트워크 | Packet |
| 데이터링크 | Frame |
| 물리 | Bit |

---

## 📌 핵심 요약 한 문장

- 물리: **신호**
- 데이터링크: **프레임**
- 네트워크: **경로**
- 전송: **신뢰성**
- 세션: **대화 관리**
- 표현: **데이터 형식**
- 응용: **의미 있는 통신**

---

## 🔑 OSI vs TCP/IP 관계

- OSI 5·6·7 계층 → TCP/IP **응용 계층**
- OSI 1·2 계층 → TCP/IP **네트워크 접근 계층**

OSI는 **사고 모델**,  
TCP/IP는 **구현 모델**이다.
