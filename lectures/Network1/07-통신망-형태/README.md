# Media Access 방식 정리 (경쟁 방식 / 토큰 제어 방식)

## 1. 경쟁 방식 (Contention-Based Access)

### 개념
- 여러 노드가 **하나의 전송 매체를 경쟁**
- 먼저 전송한 노드가 사용
- 충돌 발생 가능

### 핵심 특징
- 충돌(Collision) 가능
- 충돌 감지 또는 회피 필요
- 구조 단순
- 트래픽 많아질수록 효율 감소

### 대표 기법
- CSMA/CD
- CSMA/CA
- ALOHA / Slotted ALOHA

### 장단점
- 장점
    - 구현 간단
    - 트래픽 적을 때 효율 좋음
- 단점
    - 충돌 발생
    - 지연 시간 예측 불가

### 예시
- 이더넷(Ethernet)
- 무선 LAN(Wi-Fi)

### 비유
- 여러 사람이 동시에 말하려는 회의실
- 겹치면 멈추고 다시 말함

---

## 2. 토큰 제어 방식 (Token Passing Access)

### 개념
- **토큰(Token)**이라는 제어 신호를 가진 노드만 전송 가능
- 토큰을 순서대로 전달

### 핵심 특징
- 충돌 없음
- 전송 순서 보장
- 지연 시간 예측 가능

### 대표 기법
- Token Ring
- FDDI

### 장단점
- 장점
    - 충돌 없음
    - 공정성 보장
- 단점
    - 토큰 관리 필요
    - 토큰 분실 시 복구 필요
    - 구조 복잡

### 예시
- Token Ring 네트워크
- FDDI

### 비유
- 발언권이 있는 마이크를 돌려가며 말함

---

## 3. 핵심 비교 표

| 구분 | 경쟁 방식 | 토큰 제어 방식 |
|----|---------|--------------|
| 접근 방식 | 경쟁 | 순차적 |
| 충돌 | 발생 | 없음 |
| 제어 방식 | 자율적 | 중앙/분산 제어 |
| 지연 | 가변 | 예측 가능 |
| 효율 | 트래픽 적을 때 높음 | 트래픽 많을 때 안정 |
| 구조 | 단순 | 복잡 |
| 공정성 | 낮음 | 높음 |

---

## 4. 3초 판별법

- **충돌 얘기 나오면** → 경쟁 방식
- **토큰 얘기 나오면** → 토큰 제어
- **이더넷 / Wi-Fi** → 경쟁 방식
- **Token Ring / FDDI** → 토큰 제어 방식

# 교환 방식 정리 (회선교환 / 메시지교환 / 패킷교환)

## 1. 회선교환 (Circuit Switching)

### 개념
- 통신 전에 **전용 회선을 먼저 설정**
- 통신 종료 시까지 **회선 독점**

### 특징
- 연결 → 전송 → 해제
- 지연 거의 없음
- 회선 이용 효율 낮음

### 장단점
- 장점: 실시간성 우수
- 단점: 회선 낭비 심함

### 예시
- 전화 통화 (PSTN)

### 비유
- 전화 통화 (통화 중엔 선 독점)

---

## 2. 메시지교환 (Message Switching)

### 개념
- 데이터를 **하나의 메시지 단위로 전송**
- 중간 노드에서 **저장 후 전달(Store & Forward)**

### 특징
- 회선 설정 불필요
- 메시지가 커질수록 지연 증가
- 실시간 통신 불가

### 장단점
- 장점: 회선 효율 증가
- 단점: 지연 큼, 현재는 거의 사용 안 함

### 예시
- 초기 전보 시스템

### 비유
- 편지 (우체국에서 통째로 보관 후 전달)

---

## 3. 패킷교환 (Packet Switching)

### 개념
- 데이터를 **패킷 단위로 분할**
- 각 패킷을 **독립적으로 전송**
- 도착 후 재조립

### 특징
- 회선 설정 불필요
- 경로가 달라질 수 있음
- 지연이 가변적

### 장단점
- 장점: 효율적, 장애에 강함
- 단점: 순서 보장 필요

### 예시
- 인터넷 (IP 네트워크)

### 비유
- 택배 여러 박스 (나눠 보내고 다시 조립)

---

## 4. 핵심 비교 표

| 구분 | 회선교환 | 메시지교환 | 패킷교환 |
|----|--------|-----------|---------|
| 회선 설정 | 필요 | 불필요 | 불필요 |
| 데이터 단위 | 연속 신호 | 메시지 | 패킷 |
| 저장 후 전달 | X | O | O |
| 지연 | 거의 없음 | 큼 | 가변 |
| 실시간성 | O | X | O (보완 필요) |
| 효율성 | 낮음 | 보통 | 높음 |

---

## 5. 3초 판별법

- **미리 길을 깔아두면** → 회선교환
- **안 쪼개고 통째로 보내면** → 메시지교환
- **쪼개서 보내면** → 패킷교환
